# 进度日志 — Project_01: 股票分析工具优化

## 项目目标

将 12 个零散的股票量化分析脚本重构为规范化的 Python 包，并提供 Web 界面、自动调度、复盘回测等功能。

## 技术决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 端口 | 8765 | 避免与常见端口冲突 |
| 分类数据 | `data/sw.json` | Excel 在服务器环境不可用 |
| 推送认证 | `secrets.GITHUB_TOKEN` | Actions 内置，无需额外配置 |
| 复盘记录 | 仅自动调度写入 | 避免手动点击污染数据 |
| 新闻来源 | akshare → 东方财富 | 免费可用，无需额外 API Key |
| 关键词映射 | 硬编码字典 | 简单可靠，后续可扩展 |

## 已知问题

1. **复盘页面图表点击仍不生效** — 尝试了 Chart.js onClick 和 DOM addEventListener，用户反馈仍无法点击查看选股。可能是 chartjs-plugin-zoom 残留或 canvas 点击事件被拦截。建议：用 HTML 表格代替图表点击交互（已在表格行实现点击查看选股）。
2. **V3 策略的 K 线获取偶有超时** — 腾讯 API 有时限流。建议：增加重试机制。
3. **复盘 6.12 无数据** — 最后一天的选股没有次日涨跌幅，正常现象。
4. **新闻关键词映射不够全** — 当前只有 50+ 条规则，遇到新概念可能匹配不上。建议：定期维护 `KEYWORD_TO_INDUSTRY` 字典。
5. **本地自动调度器完成后尚未推送** — `auto_scheduler.py` 已创建但用户未确认是否需要在 GitHub Actions 之外再运行本地调度。

### 常见翻车点（不要重犯）

1. **代码改了不推送** → 所有修改最后必须 `git push`，否则 GitHub Actions 跑的是旧代码
2. **文件路径 dirname 层数** → `stock_analyzer/data/loader.py` 需要 **3 次** `dirname` 才能到项目根
3. **Windows .bat 编码** → 必须用 GBK 编码，否则 CMD 中文乱码
4. **复盘数据只显示部分** → Flask 路由的 `history[-30:]` 切片取的是最旧的数据，改为 `history` 返回全部
5. **Chart.js 插件兼容** → `chartjs-plugin-zoom` 可能导致图表初始化失败，尽量少用第三方插件
6. **序列化 pandas 对象** → pandas Series/Timestamp 不能直接 `json.dump`，必须转 Python 原生类型
7. **代码改完要自测** → 不要只靠 curl 测 API，要实际打开浏览器看看
