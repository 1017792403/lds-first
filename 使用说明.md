# 📊 Stock Analyzer — 股票量化选股工具使用说明

## 🚀 快速开始

### 启动

双击桌面上的 **`start_stock_analyzer.bat`**：

1. CMD 窗口打开，自动检查依赖（首次需安装 Flask）
2. 等待 3 秒后浏览器自动打开
3. 看到仪表盘页面即启动成功

> 如果浏览器没有自动打开，手动访问 `http://127.0.0.1:8765`

### 页面导航

| 页面 | 功能 | 入口 |
|---|---|---|
| **仪表盘** | 运行选股、查看结果、图表 | 默认首页 |
| **复盘** | 历史胜率统计、每日记录 | 顶部导航栏 |
| **回测** | 策略对比（basic vs v2） | 顶部导航栏 |
| **缓存** | 查看/清空数据缓存 | 顶部导航栏 |

## 🧠 三种选股策略

### ① 经典选股（Basic）— 🚀 推荐首选

**选股逻辑：**

```
行业分类 → 全市场行情拉取 → 行业评分 → 个股评分 → Top10
```

**行业评分权重：**

| 因子 | 权重 | 含义 |
|---|---|---|
| 行业平均涨幅 | 30% | 板块整体涨跌 |
| 上涨家数占比 | 20% | 板块赚钱效应广度 |
| 前三领涨股均值 | 25% | 板块龙头强度 |
| 总成交额 | 15% | 资金关注度 |
| 大涨(>5%)家数 | 10% | 板块赚钱效应深度 |

**个股评分权重：**

| 因子 | 权重 | 含义 |
|---|---|---|
| 当日涨幅 | 35% | 个股强度 |
| 板块内相对强度 | 30% | 在板块中是否领涨 |
| 量比 | 20% | 成交量是否放大 |
| 成交额 | 15% | 资金介入深度 |

**入选规则：**
- 根据市场强弱动态选取 **M 个行业**（牛市 20 个、震荡 12 个、熊市 6 个）
- 每行业取成交额前 15 名作为候选
- 最终 **10 只**，单行业最多 **3 只**

**适合场景：** 日常快速选股，综合判断板块轮动和个股强度。

---

### ② V2 修正版（V2）— 🔧 风险控制升级

在经典选股基础上增加 **4 个修正因子**：

**过热惩罚（Overheat Penalty）：**
```
涨幅 > 15%  → 权重 × 0.5
涨幅 > 12%  → 权重 × 0.7
涨幅 > 8%   → 权重 × 0.85
涨幅 ≤ 8%   → 权重 × 1.0
```
> 防止追高涨棒，避免买入已透支的股票

**缩量惩罚（Volume Penalty）：**
```
量比 < 1.0   → 权重 × 0.6
量比 < 1.2   → 权重 × 0.8
量比 ≥ 1.2   → 权重 × 1.0
```
> 没有成交量配合的上涨不可持续

**板块加成（Sector Boost）：**
- 评分前 5 的行业 → 板块内所有股票 × **1.2 倍**
- 强者恒强，优先选择最强板块

**温和上涨奖励（Moderate Bonus）：**
```
涨幅 2-8%      → × 1.3 倍（最佳区）
涨幅 0-2%      → × 1.1 倍
涨幅 < 0% 或 > 8% → × 0.8 倍
```
> 温和放量上涨的股票持续性最好

**入选规则：** 单行业最多 **2 只**（更分散）

**适合场景：** 震荡市、市场过热时，追求更稳健的选股。

---

### ③ V3 多因子（V3）— 🧠 事件+技术+人性

综合 **三大维度** 评分：

**技术面（40%）：**
```
MACD > 0           → +15 分
MACD 柱 > 0        → +10 分
MACD 金叉           → +20 分
KDJ 在 20-80        → +10 分
KDJ > 100          → -20 分（超买）
量比 1.5-3 倍       → +15 分
价格斜率 > 0.5      → +10 分
量价背离（看空）     → -20 分
```

**事件面（20%）：**
基于事件驱动板块（如 SpaceX IPO、长鑫科技获批等），由人工配置的事件评分（0-100）。

**人性面（30%）：**
考虑市场情绪和 FOMO（错失恐惧症）：
- FOMO 过高（>75）→ 减分（散户追涨危险）
- 温和上涨 + 放量 → 加分（健康上涨）
- KDJ 超买 → 减分

**适合场景：** 有明显事件驱动、题材炒作的市场阶段，需要结合消息面选股。

---

### 策略选择建议

| 市场状态 | 推荐策略 | 理由 |
|---|---|---|
| 普涨牛市 | **V2** | 过热惩罚避免追高，留出安全边际 |
| 结构震荡 | **Basic** | 综合评分灵活适应板块轮动 |
| 题材炒作 | **V3** | 事件驱动捕捉热点 |
| 首次使用 | **Basic** | 经典方法，易理解 |

## ⏰ 自动调度 — 每天 08:00 / 17:00 自动运行

服务器启动后，后台会自动在以下时间运行三种策略并记录结果：

| 时间 | 执行内容 | 用途 |
|---|---|---|
| **08:00** | basic + v2 + v3 各跑一次 | 开盘前获取当日选股推荐 |
| **17:00** | basic + v2 + v3 各跑一次 | 收盘后更新数据用于复盘 |

- 无需手动操作，到点自动执行
- 同一天同一时段不会重复跑
- 结果自动写入复盘历史

## 📊 可视化报告

每次运行选股后，系统会自动生成 **HTML 交互报告**，包含：

- **得分对比柱状图** — 10 只入选股票的综合得分
- **行业分布图** — 入选股票的板块分布
- **Top 10 详情卡片** — 每只股票的名称、代码、板块、得分、涨幅、选股逻辑

点击仪表盘上的 **「完整 HTML 报告」** 链接即可查看。

---

## 📋 每日复盘

系统会自动记录每次选股结果。次日开盘后可以：

1. 打开 **复盘页面**（顶部导航栏）
2. 查看历史胜率、平均涨幅等统计
3. 胜率曲线图一目了然

---

## 📉 策略回测

在 **回测页面** 点击「运行回测」：

- 系统基于当前行情数据同时运行 basic 和 v2 策略
- 对比两者的平均得分、平均涨幅、运行耗时
- 帮助判断当前市场下哪个策略更优

## 💾 缓存管理

数据缓存说明：

| 数据类型 | 缓存时间 | 说明 |
|---|---|---|
| 行业分类 | 7 天 | 申万行业分类变化不频繁 |
| K 线数据 | 1 天 | 日维度足够 |
| 行情快照 | 5 分钟 | 实时行情短期有效 |

在 **缓存页面** 可以查看状态或一键清空。

## ⚙️ 动态调参 — 在浏览器里修改选股逻辑

在导航栏点击 **「设置」**，可以看到并修改所有选股参数：

### 可调参数

| 分类 | 参数 | 说明 |
|---|---|---|
| **行业评分权重** | 平均涨幅/上涨比例/领涨强度/成交额/大涨家数 | 调整各因子对行业评分的影响 |
| **个股评分权重** | 涨幅/相对强度/量比/成交额 | 调整各因子对个股评分的影响 |
| **选股参数** | 最终推荐数/单行业上限/候选数/行业加成比例 | 调整入选规则 |
| **市场阈值** | 牛市/震荡/熊市的判断标准和 M 值 | 调整动态行业选择 |
| **V2 修正因子** | 过热惩罚阈值/缩量惩罚阈值 | 调整风险控制强度 |
| **V3 事件板块** | 各板块的事件分/曝光度 | 调整事件驱动权重 |

### 操作方式

1. 在输入框中修改数值
2. 点击 **「保存修改」**
3. 下次运行选股时新参数生效
4. 点击 **「恢复默认」** 一键还原

> 修改会保存到 `user_config.json`，重启后依然保留。

**Q: 点击按钮后等很久才出结果？**
A: 首次运行需要拉取全市场 5000+ 只股票行情，约 5-15 秒。后续运行因为有缓存会更快。

**Q: 报错 "name 'xx' is not defined"？**
A: 代码有更新，重新双击启动即可（新版本自动同步）。

**Q: 提示 "端口被占用"？**
A: 默认端口 8765，如需更换在启动前设置环境变量 `set PORT=xxxx`。

**Q: 选股结果不准？**
A: 量化选股仅供参考，不构成投资建议。建议结合基本面分析使用。
